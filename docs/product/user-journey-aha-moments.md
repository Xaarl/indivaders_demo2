# User Journey And Aha Moments

## What A Small Indie Developer Should Understand

Within 30 seconds:

- Indievaders helps with Steam positioning before demo, Next Fest, outreach, or launch.
- It is for indie teams that need better comparables and clearer next actions.
- The first paid product is an early Steam Positioning Report, not a generic analytics dashboard.

Within 60 seconds:

- The report separates true comparables from attractive but unsafe references.
- It translates review patterns and production context into decisions.
- The interactive sample report is the best way to evaluate the product.

Within 5 minutes:

- The report has evidence labels.
- Unknown public facts are not guessed.
- The paid report becomes a private workspace for one game.
- Market Watch is a future refresh layer, not the current core sale.
- The next step is either reading the sample report or requesting an early report.

## Core Aha Moments

1. The false-comparable trap

The user sees a familiar game and realizes: "I was using this as a benchmark, but the license, publisher, or scope makes it unsafe."

2. Review language becomes decisions

The user sees that nearby player complaints are not just sentiment. They become demo/page checks: prove feel early, avoid scope overpromising, clarify the hook.

3. Production reality changes the comparison

The user understands that indie-looking games can hide production support, publisher context, or content breadth.

4. Price is expectation management

The user sees price bands as promises that need visible proof, not as abstract monetization advice.

5. Evidence creates trust without slowing the report

The user can open a source drawer when they care, but the main report stays focused on decisions.

6. The workspace is where the report becomes work

The user can save comparables, mark actions done, and return to the source log.

7. Market Watch gives the reason to return

Later, the user understands that tracked comparables and review/page changes can alter recommendations over time.

## Journey Stages

### Visitor

Mindset: "I need to know if this is relevant."

Needs:

- Clear promise.
- Real examples.
- No corporate fluff.
- No overbuilt SaaS claim.

Best next action: open interactive sample report.

### Skeptical Evaluator

Mindset: "Show me something I would not catch myself."

Needs:

- Strong sample verdicts.
- Specific game traps.
- Evidence access.
- Actionable next steps.

Best next action: inspect comparables and review risks.

### Interested Buyer

Mindset: "Could this help my game?"

Needs:

- Price clarity.
- Clear intake requirements.
- What happens after request.
- Confidence that private game context stays private.

Best next action: submit early report request.

### Paid Customer

Mindset: "What should I change?"

Needs:

- Executive read.
- Specific next actions.
- Evidence labels.
- Source appendix.
- Workspace organized around their game.

Best next action: complete top actions and save/refine comparables.

### Returning Customer

Mindset: "What changed since the report?"

Needs:

- Refresh summary.
- Changed signals.
- Evidence labels.
- Recommendation deltas.

Best next action: review Market Watch changes and adjust Steam/page/demo decisions.

## Value Moments To Design Around

- "That reference is useful, but not for the reason I thought."
- "This review risk maps to something I can fix before demo traffic."
- "I should not copy that game's scope just because it looks indie."
- "My price implies proof my page does not show yet."
- "The report is opinionated, but I can see what is fact, reported, estimated, and inferred."
