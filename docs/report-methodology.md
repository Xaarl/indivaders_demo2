# Report Methodology

Operational summary. The richer evidence and trust source of truth lives in `docs/product/trust-evidence-model.md`.

The Steam Positioning Report is evidence-first. It is designed to help an indie team make better Steam-page decisions without pretending public data is more precise than it is.

## Public Sample Boundary

The free sample brief is short enough to read in under 2 minutes. It is a teaser of the decision logic, not a full report. It may show:

- an executive read;
- 2-3 true comparable examples;
- one production reality example;
- one review risk excerpt;
- one price/scope sanity example;
- 3 next actions;
- a small source/confidence note.

The paid report is separate and private.

## Paid Report Sections

The paid Steam Positioning Report includes:

- Executive Read
- Market Lane
- True Comparables
- Production Reality
- Tag And Positioning
- Review Risk Map
- Price And Scope Sanity
- Launch Timing
- Creator Starter List
- Prioritized Next Actions
- Source Appendix

## Evidence Labels

Use exactly these labels:

- `Confirmed`
- `Reported`
- `Estimated`
- `Inferred`
- `Not publicly confirmed`

Definitions:

- `Confirmed`: visible in a primary public source such as Steam, an official site, a press kit, or a publisher page.
- `Reported`: stated by a developer, publisher, interview, article, talk, or other public account.
- `Estimated`: provided by a third-party estimator or calculated from public signals with uncertainty preserved.
- `Inferred`: reasoned from multiple public signals; useful for positioning, not treated as fact.
- `Not publicly confirmed`: unavailable in reliable public sources; never filled with a guess.

## Evidence Types

Use exactly these evidence types:

- `steam_store`
- `steam_reviews`
- `official_site`
- `press_kit`
- `developer_interview`
- `postmortem`
- `publisher_page`
- `public_database`
- `article`
- `reddit_ama`
- `youtube_or_podcast`
- `third_party_estimate`

## Unknown Data Rules

Unknown budget, team, sales, revenue, and marketing data must be represented as `Not publicly confirmed`, not left blank and not replaced with a guess.

Third-party sales or revenue estimates can appear only as `Estimated`, with the estimate type made clear. They should never be written as confirmed truth.

Every recommendation needs either evidence or a clear inference label. If a public source cannot support a claim, the report should say so plainly.

## Case File Rules

Public case teasers use one naming system: real game case files. They are concise landing-page examples, not full analyses.

The three current case files are:

- Real Game Case File 001: `TMNT: Shredder's Revenge` as a licensed benchmark trap.
- Real Game Case File 002: `Balatro` as an outlier pattern case.
- Real Game Case File 003: `Dave the Diver` as an indie-looking but not indie-scale case.

TMNT should not be framed as a small indie benchmark. It can teach beat-'em-up readability, but the license and publisher context make it unsuitable as a direct small-team benchmark.
