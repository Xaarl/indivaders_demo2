# Designer Brief

Date: 2026-06-16

## Objective

Make Indievaders feel like one coherent product: a signal intelligence instrument for Steam positioning. Preserve the memorable invader/black-hole hero, then make the rest of the experience calmer, sharper, and more operational.

## Product Context

Indievaders helps serious solo indie developers and small teams avoid bad Steam positioning decisions before demo, Next Fest, outreach, pitch, or launch.

The first sellable product is:

```text
$49 early Steam Positioning Report
```

The product ladder is:

1. Landing.
2. Interactive sample report.
3. Early report request.
4. Paid project-specific report.
5. Private project workspace.
6. Future Market Watch.

## Surfaces To Design First

1. Landing hierarchy.
2. Real-game case diagnostic console.
3. Interactive sample report overview.
4. Evidence/source chip system.
5. Early report intake grouping.
6. Workspace preview status treatment.

## Keep

- Existing invader/black-hole hero direction.
- Real-game case file direction.
- Interactive report as the proof artifact.
- Work-grade private workspace direction.
- Evidence/source access as a trust signal.

## Avoid

- Generic dark SaaS dashboards.
- Full arcade maximalism below the hero.
- Long methodology pages in the UI.
- Fake live Market Watch.
- Fake account or persistence cues.
- Raw data explorer visual language.
- AI assistant framing.

Use the component direction from:

- `docs/design/07-redesign-priorities.md`

- `DecisionRow`

- `SeverityChip`

- `ComparableMap`
- `ActionQueue`
- `SourceDrawer`

## First Design Deliverables

Deliver these before broad visual exploration:

1. Landing hierarchy pass with one primary path: `#sample-report` then `#order-report`.
2. Case diagnostic console wireframe or mockup.
3. Sample report overview mockup with one lead decision plus supporting decision rows.
4. Evidence/source chip treatment.
5. Workspace preview command strip and local-preview status treatment.
6. Mobile hierarchy notes for landing and report overview.

## Acceptance Criteria

- A new visitor understands the product path in under one minute.
- The sample report clearly catches one wrong-benchmark decision.
- Case cards teach the trap before click-through.
- Evidence labels are visible but not noisy.
- Workspace preview does not imply backend persistence.
- Market Watch is clearly future.
- The design stays game-adjacent but serious enough for paid report trust.
